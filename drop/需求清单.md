# 落字 / DropText — 需求清单

> 屏幕上的字，落下来，留住
> Words on screen, dropped and kept
> 零权限、零封号风险、通用所有App
> 独立程序，不属于任何项目

---

## 核心逻辑

**你刷屏幕 → 屏幕出现文字 → 自动捕获 → 去重 → 存本地**

不登录任何平台、不调API、不读App数据。

---

## 技术方案

### 阶段一：Chrome扩展 — 录制+自动滚屏

- 在当前网页上下文触发录屏（不用popup，避免失焦中断）
- 自动寻找聊天滚动容器（虚拟列表+普通滚动都支持）
- rAF像素级匀速上滚（不用smooth，避免运动模糊和竞争条件）
- 检测到顶/到顶文字标志自动停止
- 懒加载时暂停等待
- 视频通过Base64传给background安全下载

### 阶段二：Python脚本 — OCR提取

- 逐帧截图+Tesseract OCR
- 滑动窗口去重（LCS匹配相邻帧重叠区域）
- 输出jsonl+md双格式

---

## 核心难点及解法

### 1. 去重
- 同一段话滚回来再看不能记两遍
- 内容哈希去重 + 滑动窗口LCS

### 2. 识别谁说的
- DOM模式：气泡左右对齐
- OCR模式：气泡位置
- 识别不了标"未知"，整理时一眼能分

### 3. 图片/表情/语音
- 标记 [图片] [表情] [语音]
- 可选截取图片区域保存

### 4. 虚拟滚动
- transform: translateY检测位移变化
- scrollTop失效时用DOM状态指纹做双保险

### 5. 懒加载
- 检测loading骨架屏/skeleton/spinner
- 加载中暂停滚动，渲染完继续

### 6. 起步死
- 前90帧(约1.5秒)跳过到顶检测
- 启动时先滚到底部

## 输出格式

```
screen-record/
├── record.jsonl      # 逐条，实时追加
├── readable.md       # 可读版
└── media/            # 截取的图片
```

## 使用流程

1. 打开聊天App网页版
2. 点扩展 → 开始录制+滚屏
3. 等它自动滚到顶停止
4. 下载的视频用ocr_extract.py提取文字
